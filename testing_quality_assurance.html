Test Coverage Example:

// Unit Test Example
describe('ShoppingCart', () => {
  it('should calculate total with tax', () => {
    const cart = new ShoppingCart();
    cart.addItem({ price: 100, quantity: 2 });
    cart.addItem({ price: 50, quantity: 1 });
    
    expect(cart.getSubtotal()).toBe(250);
    expect(cart.getTotalWithTax(0.08)).toBe(270);
  });
  
  it('should apply percentage discount', () => {
    const cart = new ShoppingCart();
    cart.addItem({ price: 100, quantity: 1 });
    cart.applyDiscount('SAVE20', 0.20);
    
    expect(cart.getTotal()).toBe(80);
  });
});

Testing Best Practices: The Golden Rules

Testing Commandments

📝 Test Names Should Tell a Story: test_user_can_checkout_with_valid_credit_card

🎯 One Assertion Per Test: Each test should verify one specific behavior

🔄 Tests Should Be Independent: Order shouldn't matter, no shared state

⚡ Fast Tests Get Run More: Mock external dependencies in unit tests

🤖 Automate Everything Possible: If you do it twice, automate it

📊 Measure Code Coverage: But remember 100% coverage ≠ bug-free

🎭 Test Happy Path and Edge Cases: What happens with null, "", -1?

🔍 Test Data Should Be Realistic: Use production-like data sets

🚨 Failing Tests Should Be Clear: Good error messages save debugging time

🔧 Maintain Your Tests: Delete obsolete tests, refactor along with code
